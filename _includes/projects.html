starmos.io

Inspired by flat design, starmos.io is an online web game created using node.js. The objective of the game is to explore the map from a selection of spaceships, and find asteroids, comets, or enemy players in order to unlock new abilities. All graphics were designed using simple shapes and colors, some of which are generated during gameplay.

IPMP Data Exploration Interface

This app is built with the ArcGIS API and Web AppBuilder Source. The interface presents metadata in the form of map layers, chart infographics, and other derived overlays for use by local government agencies in Iowa. The app includes various tools to estimate costs for road treatments, and to save a selection of data as a geodatabase or shapefile through the download geoprocessing service.

Lab Assistant

Lab Assistant is a companion app for teachers to use in the classroom. Designed for Android, users can track attendance, assign homework, organize students into groups, and manage other details of their classroom or lab. Students can use the app to chat with other users about class material, and notify the instructor for help. Data is served to the app from a MySQL backend, accessible via a REST API.

Roguelike Game

A roguelike game written in C & C++ that runs inside the terminal, with graphics managed by ncurses . Maps are randomly generated: hallways are marked with '#', rooms as '.', enemies as alpha characters, and items as special characters. The player is controlled via keyboard input, and must navigate through the dungeon to find enemy characters and collect items. The game ends once either the player or all enemies are defeated.

ARBnb

ARBnb is an augmented reality application developed for Android, and developed for LEAN Techniques as part of my senior project. Using Google's ARCore library in Unity, the app presents an interface for 3d spaces: interactive tooltips and icons are augmented into the room via the mobile camera, displaying property information to the user, such as public hiking trails or room info.

LEGv8 Emulator

A subset of ARMv8's Instruction Set Architecture, LEGv8 is a reduced form of assembly with standard commands such as 'ADD', 'LSL', and 'LDUR'. Assembly is passed into this LEGv8 Emulator, which parses and executes each statement. Memory is explicity allocated into static arrays for registers (X9, X10, X11...) and variables (e.g. X0 after executing the above 'LDUR' statement). The emulator is written completely in C, and was created as a class project for COM S 321: Introduction to Computer Architecture and Machine-Level Programming.
